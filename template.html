dcd — duplicate code detector

This report shows code duplication across your codebase. It is generated by dcd, a tool that identifies duplicate or near-duplicate code blocks by comparing normalized line hashes across all files.

Heatmap view

The main grid is a file × file matrix. Each cell's color represents how much code two files share — darker blue means little overlap, red means heavy duplication. The diagonal is always dark (a file compared to itself is excluded). Hover over any cell to see the file pair and duplication percentage.

Scatter plot view

Click any heatmap cell to drill down into a Ducasse-style dot matrix for that file pair. Each dot means the line at that row in file A matches the line at that column in file B. Diagonal runs of dots indicate copied or moved blocks of code. Scattered dots are common boilerplate (braces, blank lines, imports).

Sidebar

Lists all files sorted by their highest duplication percentage against any other file. Click a file to highlight it.

Controls

sort: name / sort: duplication — reorder the heatmap axes.
Ctrl+scroll — zoom in/out on the heatmap.
+ − ⊡ — zoom controls (bottom-right).
Scroll to pan when zoomed in.
← back to heatmap — return from scatter view.

Header stats

files — total files scanned. lines — total source lines across all files. dup lines — total duplicate lines detected.
